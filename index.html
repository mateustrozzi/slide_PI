Arquitetura do Layout

Distribuição de espaço no HEADER

1. header-left (flex: 1): Container do logo. No Desktop, equilibra o lado esquerdo para forçar a centralização do meio.
2. nav-container (flex: 2): Centro do cabeçalho com justify-content: center para alinhar os links de navegação.
3. header-right (flex: 1): Container para o botão CTA e o ícone do menu, garantindo a simetria perfeita.

Tags e Lógica de Implementação

  • SVG Inline: Inserido direto no HTML, permitindo a animação do logo em svg, usando a tecnologia SMIL (Synchronized Multimedia Integration Language).
  • Checkbox Hack: Foi usado um hack em CSS para criar, exibir e ocultar o menu hambúrguer, usando o checkbox.
  • Pseudo-elemento ::before: No mobile, atua de forma invisível à esquerda para empurrar o logo e centralizá-lo perfeitamente na tela.

Comportamento Responsivo

Performance e Responsividade

  • <picture> e <source>: Otimização de banda carregando arquivos específicos para Mobile, Tablet e Desktop.
  • Unidade 100svh: Resolve o problema clássico do mobile onde a barra de endereços "quebra" o layout de altura total.
  • Filtros de Saturação: No Desktop, aplicamos filtros dinâmicos via CSS para intensificar as cores das fotos sob o gradiente.

Seção de Serviços

Estrutura de cards focada em semântica estrita, acessibilidade e flexibilidade de layout.

<div class="servicos">: Divisão genérica. Escolha deliberada já que o card possui um propósito puramente visual, sem significado semântico isolado próprio.
<img> com alt: Tag semântica obrigatória para imagens de conteúdo real (essencial para acessibilidade). Evitou-se o uso de background-image porque as fotos não são meramente decorativas.
<i class="fa-solid">: Convenção padrão da biblioteca Font Awesome. Utiliza a tag <i> por ser semanticamente neutra para a renderização de ícones.

Composição do Elemento Visual

Técnica de sobreposição para fazer o ícone flutuar na borda da imagem:

.img-wrapper

Configurado com position: relative.

Atua estritamente como o "pai" que cria o contexto de posicionamento limitador na tela.

i.fa-solid (Ícone)

Configurado com position: absolute.

Utiliza valores negativos de bottom e right para "escapar" dos limites do container e flutuar com precisão.

Layout Adaptável e Acabamento

Inversão de Direção

Uso estratégico de Flexbox mudando de flex-direction: column no mobile para row no desktop.

Permite reordenar ou realinhar a exibição dos cards na media query sem qualquer duplicação de código HTML.

Ajuste de Borda

Aplicação cirúrgica de border-radius: 10px 10px 0 0 diretamente na tag da imagem no desktop.

Arredonda exclusivamente as quinas superiores, garantindo encaixe perfeito no topo do card e eliminando o risco de linhas fantasmas.

Seção Empresa

Layout focado em storytelling visual, otimização de espaço e separação de conteúdo.

<div id="lado-esquerdo">: Wrapper estratégico que agrupa todo o bloco de texto e ícones. No desktop, sua função é receber width: 50% para delimitar perfeitamente a área de leitura.
<p id="sobre-nos">: Atua como um rótulo de categoria (tagline) posicionado logo acima do título principal. Usa a tag de parágrafo para isolar o elemento em um bloco autônomo e altamente estilizável.
<hr> (Linha Decorativa): Tag semântica de quebra temática, mas aqui totalmente customizada via CSS. Teve sua borda padrão zerada para se transformar em uma linha decorativa colorida moderna, sem necessidade de JS ou divs extras.

Abordagem Responsiva da Imagem

Gerenciamento inteligente da foto da fachada da empresa com base no dispositivo:

No Mobile

#imagem { display: none; }

A foto é completamente ocultada em screens pequenas para economizar espaço vertical e evitar sobrecarga visual.

No Desktop

width: 50vw; height: 100vh;

A imagem ganha vida ocupando exatamente metade da largura da janela de visualização (vw) e a altura total da tela (vh).

Refinamento Visual via CSS Puro

Duas soluções que eliminam o peso de imagens editadas externamente:

object-fit: cover

Garante que a foto da fachada preencha todo o seu espaço determinado de 50vw/100vh sem sofrer distorções ou achatamentos.

O navegador calcula o redimensionamento automaticamente e corta as bordas excedentes, se necessário.

::before + linear-gradient

Cria uma camada de efeito por cima da foto (indo do preto opaco ao transparente) gerada direto pelo navegador.

Essa técnica elimina a necessidade de exportar uma imagem modificada no Photoshop, reduzindo o consumo de banda.

Seção Diferenciais

Estrutura focada em escalabilidade de código e eliminação de elementos HTML puramente decorativos.

<div class="cards"> (Classe Compartilhada): Aplicada uniformemente nos 4 cards da seção. Essa abordagem permite o reaproveitamento total da folha de estilos (DRY), garantindo consistência visual e um arquivo final leve.

Estratégia de Layout Híbrido

Transição inteligente entre bidimensional e unidimensional conforme o dispositivo:

Mobile: CSS Grid

grid-template-columns: 1fr 1fr;

Cria uma malha de duas colunas iguais. A unidade fracionária (1fr) distribui o espaço disponível de forma proporcional automática.

Desktop: Flexbox

display: flex;

Substitui o Grid na media query porque o layout vira uma única linha horizontal (unidimensional), oferecendo maior controle de alinhamento.

Seletores de Posição Estratégicos

Desenho de linhas divisórias diretamente por lógica de folha de estilo:

:nth-child(odd) e :nth-child(1), :nth-child(2)

Em vez de poluir o código HTML adicionando tags vazias para renderizar as linhas de separação dos cards, o projeto mapeia a posição exata de cada elemento na árvore estrutural.

Benefício: O HTML permanece 100% limpo, e as linhas divisórias se adaptam ou desaparecem sozinhas no desktop.

Contato & FAQ

Integração de elementos interativos, mapas embutidos e acessibilidade nativa sem dependências.

<button class="perguntas">: Uso de tag nativa para os gatilhos do FAQ. Garante acessibilidade total por teclado de forma automática, eliminando a necessidade de hacks em divs.
<iframe> (Google Maps): Embed oficial que entrega um mapa dinâmico e 100% interativo, rodando de forma isolada sem consumir processamento do script principal.

Layout Flexbox Tridimensional

#contato { display: flex; flex-wrap: wrap; height: 100vh; }

Garante altura útil do monitor (100vh) no desktop, com quebra automática se necessário.

Coluna FAQ

flex: 0.6;

A área de perguntas e respostas assume uma proporção ligeiramente menor do container.

Coluna Informações

flex: 1;

A área institucional e mapa ganha maior relevância. O Flexbox calcula a divisão sem porcentagens engessadas.

Manipulação Inteligente do DOM

A mecânica por trás do efeito sanfona (Accordion) do FAQ:

Navegação Relativa

nextElementSibling

O JS encontra a resposta mapeando o irmão direto do botão. Dispensa o uso de IDs individuais.

Transição de Estados

transform = 'rotate(180deg)'

O script injeta a rotação. A propriedade transition no CSS isola a responsabilidade da animação na GPU.

Ciclo de Vida e Eventos de Janela

ajustarTexto() + evento resize

Monitora o tamanho da janela e troca strings longas do desktop por versões compactas no mobile, eliminando duplicação de HTML.

Atributo defer no <script>

Garante execução do script apenas após a montagem completa da árvore do DOM, prevenindo erros de "elemento não encontrado" (Null).
